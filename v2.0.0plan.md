# uaip v2.0.0 計畫 — Pages→Workers、TypeScript、公開作品集化

> 狀態：已定案、待執行（2026-07-15 拍板）。執行慣例比照 v1：一階段一 commit、中文訊息。
> 開工前置：`git branch v1 && git push origin v1`（v1 塞進分支，main 開始 v2）。

## Context（為什麼做）

v1.0.0（2026-07-15 tag）是一個工程紀律優秀但規模小、且**不能公開**的私有專案：git 歷史裡有舊 LOGS_TOKEN、個人聯絡方式與 email 寫死在程式裡、SSR 有一個 nonce 蓋章造成的 stored-XSS 路徑、OAuth／SSR／admin 寫入 API 零測試。目標：升級成可放履歷的頂級公開 repo——平台遷移到 Workers（解鎖 DO 原子限流、cron 備份/聚合/告警）、全面 TypeScript strict、補測試、成本記帳、Demo 模式，最後洗歷史轉公開。

**已拍板決策（不再重議）**：台灣為主（中文註解與 commit 保留、README/ADR 雙語）；遷移 Workers（DNS 已驗證 uaip.cc.cd 是帳號內 zone，NS 在 Cloudflare，自訂網域可直接掛）；全面 TS strict；VPN 公開但文件低調；filter-repo 洗歷史保留全部 commit；**完工才公開**；MIT；/api-docs 改公開；新功能＝成本記帳＋Telegram 告警＋Demo 模式＋「站長」→「管理員」；品質＝ESLint/Prettier/gitleaks/CodeQL＋Playwright E2E＋OpenAPI 3.1。不做：BYOK、k6、Queues（免費層無）、圖片搬 R2。

**全程免費層**（已查證）：Workers 100k req/日＋10ms CPU、靜態資產請求免費、SQLite DO 免費（100k/日、5GB、免儲存費）、cron 免費（≤5 條）、D1/R2 免費額度照舊。

- `lib/site.js:7-8` CANON/BRAND → `siteOrigin(env, request)`：優先 `env.SITE_ORIGIN`（wrangler `[vars]`），退 `url.origin`。呼叫面：`lib/pages.js`、`functions/feed.js`、`functions/sitemap.js`、`test/helpers.js`。

### Phase B — Stored-XSS 修復＋威脅模型更新（M）
問題：`lib/site.js:105` `html()` 用 regex 對 body 所有 `<script` 蓋 CSP nonce；`lib/pages.js:141`、`functions/p/[slug].js:27` 的 `marked.parse()` 輸出未消毒 → 文章夾 `<script>` 會被 nonce 放行（管理員 token 失竊時的 stored XSS）。
- 新 `lib/sanitize.js`（零依賴白名單消毒，~100 行）：標籤/屬性/URL scheme 白名單、剝 `on*` 與 script/style/iframe。所有 `marked.parse` 呼叫點套用。
- `html()` 改只對外殼標記的 `<script data-nonce>` 蓋章（防禦縱深）。
- 更新 `docs/THREAT-MODEL.md`。
- 驗證：新增 `test/unit/sanitize.test.js`（XSS 語料）＋整合測試（發文含 payload → SSR 無 nonce-blessed script）。

### Phase C — 測試補洞：遷移前的安全網（L，零執行期變更）
用現有 `makeCtx` 模式補齊審計缺口：OAuth 全流程（fetchMock Google endpoints）、`_middleware.js` shouldLog 矩陣、全部 SSR 頁 smoke（狀態碼/CSP nonce/noindex）、feed/sitemap/img、全部 admin 內容寫入 API、account key/vpn-token、公開讀 API。
- 驗證：174 → 約 260+ 測試，對照審計清單逐項勾銷。

### Phase D — Pages → Workers 遷移（XL，核心）
- 開頭 30 分鐘 spike：空 worker 驗 `run_worker_first` 陣列、assets `_headers` 支援、SQLite DO（wrangler ~4.44 鎖版組合）。
- 新 `src/index.ts`（`export default { fetch }`）、`src/router.ts`、`src/routes.ts`（靜態 import ~50 個 handler，掛既有 `onRequestGet` 等——**functions/ 檔案原地不動**，同一 commit 可回滾部署回 Pages）。
- Router 設計：路由表陣列、註冊順序即優先序；`[id].js`→`:id`、`[[path]].js`→`*path`（餘段為 string[]，與 Pages `params.path` 同形，relay 零改動）；線性掃描；middleware 用組合函式（visitLog 移植＋新增全域 errorBoundary→reportError＋500）；無匹配→`env.ASSETS.fetch`（SPA fallback 吃 /ip /ua）。
- `wrangler.toml` 改 Workers 形：`main`、`[assets] directory="public" binding="ASSETS" not_found_handling="single-page-application" run_worker_first=["/"]`；舊設定另存 `wrangler.pages.toml`（回滾用）。
- `vitest.config.mjs` 換 Workers 模式（pool-workers 直接讀 wrangler.toml；此切換為第一子步驟、獨立驗綠）；新增 `SELF.fetch` 全路由 smoke。
- `package.json`：dev/deploy 指令換 `wrangler dev`/`wrangler deploy`；刪空的 `public/_redirects`；`public/_headers` 保留（spike 若發現不支援→worker 補標頭 fallback）。
- **割接 checklist**（非 commit）：`wrangler deploy` 上 workers.dev（綁正式 D1，期間凍結 admin 變更）→ `wrangler secret put` 四把（Pages secrets 不轉移）→ workers.dev 冒煙（匿名頁/SPA fallback/302/Bearer API/relay/vpn 隱形）→ 低流量時段 Pages 摘 domain、Worker 掛 `uaip.cc.cd`（同 zone、憑證分鐘級）→ 全功能手測（Google 登入、SSE、relay 串流、admin）→ 回滾路徑＝domain 掛回 Pages。
- 驗證：全測試綠、割接 checklist 全勾、errlog 觀察 24h。

### Phase E — Lint/Format/掃描 CI（S）
ESLint flat config＋typescript-eslint（為 F 預備）＋Prettier（全 repo 一次格式化＝**獨立 commit**＋`.git-blame-ignore-revs`）；CI 加 lint job＋gitleaks action。CodeQL **留到 Phase O**（公開 repo 才免費）。CI 仍不部署。

### Phase F — TypeScript 全面遷移＋目錄重整（L）

- 目錄重整：`functions/`→`src/routes/`、`lib/`→`src/lib/`（vendor 照搬）；文件路徑同步。前端 `public/assets/*.js` 不轉。
- 紀律：**轉換禁改邏輯**；中文註解保留；測試檔一併轉（可裁剪）。
- 驗證：`tsc --noEmit` strict 全綠、全測試綠、部署後手測一輪。

### Phase G — 全域改名 站長→管理員（S）

### Phase H — Durable Object 限流器（M）＋ADR-0007
- 新 `src/do/rate-limiter.ts`（SQLite-backed，RPC `check({svc,perMin,perDay})`）：分鐘限流＝兩桶加權滑動窗；日配額＝`d:<UTC日>:<svc>` 鍵懶重置；分片 `idFromName("u:"+user.id)`。limit 由呼叫端算好傳入，DO 只管原子計數。
- **語意升級（ADR 記載）**：修掉 D1 COUNT+放行的並發競態——原子性才是換 DO 的核心理由。
- `checkQuota` 對外契約不變（relay/chat handler 零改動）；三層降級：DO 壞或 `quota_do='0'`→退 D1 COUNT 路徑→再壞→放行（配額永不弄掛服務）。
- wrangler.toml 加 DO binding＋`new_sqlite_classes`。
- 驗證：`runInDurableObject` 單測（滑動窗數學、UTC 重置、`Promise.all` 並發恰好 limit 發過）＋降級三層整合測試。

### Phase I — Cron 派工：備份/聚合/保留/Telegram 告警（M）
- `src/cron.ts`：`"*/5 * * * *"`→tgAlertScan；`"17 19 * * *"`→rollupUsageDaily＋backupToR2＋purge（req_log 90d、sessions、pg_messages 360d）。每 job try/catch 隔離、結果寫 settings `cron_last_*`、失敗進 errlog（告警自然涵蓋 cron 自身故障）。
- tgAlertScan：settings `tg_cursor` → errlog 增量 → Telegram sendMessage（secrets `TG_BOT_TOKEN`/`TG_CHAT_ID`，未設＝no-op）→ 成功才推 cursor。
- rollupUsageDaily：migration `0003_v2_usage_daily.sql`，`INSERT OR REPLACE` 結算昨日、冪等。
- backupToR2：逐表游標分批 → JSONL → R2 `BACKUPS`（`media.data` BLOB 排除，防 10ms CPU）；保留 14 份。
- `lib/quota` 的 1% 隨機清舊列 hack 退役。DEBT.md 劃掉 #2/#4/#6/#11。
- 驗證：scheduled handler 直呼測試、Telegram fetchMock 斷言 payload 與 cursor、備份內容斷言、rollup 冪等；上線後 dashboard 手觸發＋隔日檢查 R2。

### Phase O — 洗歷史、公開、發佈（S，程序為主；**此後永不再改寫歷史**）
1. 輪替線上 LOGS_TOKEN（指令在 ADMIN.local.md）。
2. `git clone --mirror` 離線備份 → 乾淨 clone → `git filter-repo --replace-text`（replacements.txt 放 repo 外、只洗舊 token、用完即刪）。
3. 驗證：`git log --all -S '<舊token>'` 空、commit 數對、`git tag` v1.0.0 在、中文訊息＋co-author trailer 完整。
4. 重建 `v1` 分支（v1 末尾 commit 的改寫後 hash）→ `git push --force origin main v1 'refs/tags/*:refs/tags/*'` → 本機工作目錄整個重 clone。
5. GitHub 轉 Public → 開 secret scanning＋push protection → 補 CodeQL workflow（公開後免費）→ topics/description/social preview → tag `v2.0.0`＋雙語 Release notes。

## 不做清單（護欄）
零 runtime 依賴不動搖（router／sanitizer 手寫，npm 只進 devDependencies）；不用 hono/框架；不用 Queues；不做 BYOK/k6；圖片不搬 R2（DEBT #1 門檻未到）；CSP style-src 維持（ADR-0004 補裁決）；前端資產不 TS 化不打包；CI 永不部署；個人聯絡方式永不回到程式碼；Phase O 之後永不改寫歷史。

## 驗證總則
每階段：`npm run checks`（tsc＋vitest）全綠 → 本機 `wrangler dev` 手測該階段功能 → `npm run deploy` → 線上冒煙（/api/health、errlog 無新異常）。Phase D 另有割接 checklist；Phase O 另有洗史驗證清單。最終狀態：260+ 單元/整合測試＋E2E 綠、TS strict、公開 repo、demo 免登入可玩、README 有真實數據。
